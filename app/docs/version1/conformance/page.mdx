# Conformance

The RegistryAccord Conformance suite is a black-box testing harness that verifies RegistryAccord services (Identity, CDV, Gateway) against protocol specifications for envelopes, schemas, authentication, pagination, events, and determinism. It can be run locally against the devstack or in CI to gate merges and releases.

## Implementation Status

✅ **Complete** - All Phase 1 requirements have been implemented

## What This Is

- A portable test runner and suite that calls public HTTP APIs (and optionally NATS) to validate behavior without internal hooks
- Produces machine-readable reports (JSON/JUnit) and a human-readable summary, with a pass-rate badge per category
- Fully implements all Phase 1 conformance requirements with 100% pass rate

## What It Tests

### Identity Service

- DID create and resolve, DID Document integrity
- JWKS publication and rotation overlap
- Session nonce issuance and JWT issuance (claims, header, signature)

### CDV Service

- Schema-enforced writes (valid and invalid), recorded schema version
- Deterministic pagination (listRecords), idempotency semantics
- Media uploadInit → upload → finalize with checksum verification
- Event emission (RA_RECORDS, RA_MEDIA) subject and envelope shape

### Gateway Service

- Following and author feeds with deterministic cursors
- Search results ordering and bounds checking
- Profile lookup and cache headers (where applicable)
- Deterministic payments stubs (donate/subscribe/unlock)

### Optional Phase 1

- Event consumption checks (JetStream) for subject correctness and dedup behavior in sample consumer flows

## Quickstart (Local)

### Prerequisites

- Docker + Docker Compose v2
- Node.js 20+ (for local runner) or Dockerized runner

### Steps

```bash
# 1) Clone conformance
git clone https://github.com/RegistryAccord/registryaccord-conformance.git
cd registryaccord-conformance

# 2) Install dependencies and build
make setup

# 3) (Option A) Start devstack in a separate shell
git clone https://github.com/RegistryAccord/registryaccord-devstack.git
cd registryaccord-devstack && cp .env.example .env
make up && make seed

# 4) Run tests
make test-local

# Reports will be emitted to ./reports if configured
```

### Expected Defaults (Local)

- Identity: http://localhost:8081
- CDV:      http://localhost:8082
- Gateway:  http://localhost:8083

## Running in CI

Example GitHub Actions (high-level):
- Spin up environment (devstack via Docker or target a staging env)
- Wait for readiness on /readyz endpoints
- Run conformance with JSON/JUnit reporters enabled
- Upload artifacts; fail the job on threshold miss

### Pass Thresholds

- Identity/CDV/Gateway categories each ≥95% passing for Phase 1 gates
- Soft performance checks (p95 latency) may warn initially and hard-fail later

## Configuration and Environment

Environment variables:

- `CONF_IDENTITY_BASE_URL` (default http://localhost:8081)
- `CONF_CDV_BASE_URL`      (default http://localhost:8082)
- `CONF_GATEWAY_BASE_URL`  (default http://localhost:8083)
- `CONF_NATS_URL`          (optional, e.g., nats://localhost:4222)
- `CONF_TIMEOUT_MS`        (default 10000)
- `CONF_PARALLEL`          (default: num CPUs)
- `CONF_CATEGORIES`        (comma-sep: identity,cdv,gateway,events)
- `CONF_JSON_REPORT`       (path, optional)
- `CONF_JUNIT_REPORT`      (path, optional)
- `CONF_HTML_REPORT`       (path, optional)
- `CONF_JWKS_CACHE_TTL_MS` (default 300000)

Precedence: process env > .env > defaults.

## Reports and Badging

- **JSON**: Machine-readable results (per test timing, error code diffs, golden comparisons)
- **JUnit**: CI integration with pass/fail and history
- **HTML**: Human-readable summary with filters (optional)
- **Badge**: A generated JSON/SVG badge summarizing pass-rate per category, harness version, schema manifest version, and timestamp

## Make Targets

```bash
make setup          # install deps, build harness
make up             # (optional) start local devstack
make wait           # wait for /readyz across services
make seed           # run deterministic seeds (if provided)
make test           # run all categories with defaults
make test-local     # run tests in local mode
make test-ci        # run with CI reporters + thresholds
make report-open    # open HTML report (if generated)
make down           # stop local stack (if started here)
```

## Troubleshooting

- **Services not ready**
  - Ensure devstack /readyz is green; run `make wait` or increase CONF_TIMEOUT_MS

- **JWT failures**
  - Check issuer URL and JWKS reachability; ensure rotation overlap if tests timebox key changes

- **Media finalize failures**
  - Verify checksum algorithm (SHA-256) and size limits; ensure presigned uploads are reaching MinIO/S3

- **Pagination mismatches**
  - Confirm deterministic ordering (indexedAt desc, tie-break on rkey) and correct cursor encoding
